# 三.elasticsearch基本操作

## 3.1 ES结构

3.1.1 索引Index

![1598424035711](06_Redis%E7%9A%84%E7%BB%93%E6%9E%84.assets/1598424035711.png)

> ES的服务中,可以创建多个索引
>
> 每一个索引默认被分成5片存储.
>
> 每一个分片都会存在至少一个备份分片
>
> 备份的分片默认不会帮助检索数据,当ES检索压力特别大的时候,备份分片才会帮助检索数据.
>
> 备份的分片必须放在不同的服务器中.
